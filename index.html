独立开发者常用的研发工具集合

Michael Song 发布于 2025-04-07 收录于 Categories

对于独立开发者而言，选择高效、轻量且成本可控的工具是提升效率的关键。以下是根据研发流程分类的工具推荐，涵盖从需求管理到部署运维的全链条：

1.项目与任务管理

  1. Trello https://trello.com/
  • 特点：可视化看板，拖拽式任务管理，支持个人免费版。
  • 适用场景：简单任务规划、敏捷开发流程。 2禅道
  • 特点：开源免费，集成需求、测试、缺陷管理，适合中小团队。
  • 适用场景：研发全流程管理，适合注重本地化部署的开发者。 3
  • Notion：文档管理、知识库、项目看板一体化。

2. 研发

2.1 代码托管与协作

  1. GitHub
  • 特点：免费私有仓库，集成 Pull Request、Issue 追踪，生态丰富。
  • 适用场景：代码版本控制、团队协作（适合个人或小团队）。
  1. GitLab
  • 特点：免费版支持 CI/CD、代码审查，功能全面。
  • 适用场景：全栈开发，追求一站式解决方案。

2.2 开发与测试工具

  1. Visual Studio Code
  • 特点：轻量强大的 IDE，支持插件扩展，跨平台。
  • 适用场景：多语言开发、远程协作。
  1. Postman
  • 特点：API 测试与管理工具，支持自动化测试。
  • 适用场景：接口开发与调试。
  1. JUnit/Selenium
  • 特点：单元测试（JUnit）和自动化 UI 测试（Selenium）。
  • 适用场景：确保代码质量，减少手动测试成本。

2.3、用户认证与支付

  1. Supabase
  • 特点：开源 BaaS 平台，提供用户认证、数据库等功能，免费额度高。
  • 适用场景：快速构建用户系统，适合中小项目。
  1. Stripe
  • 特点：国际支付处理，支持订阅模式，开发友好。
  • 适用场景：跨境业务收款。
  1. KodePay
  • 特点：国内开发者友好，支持信用卡、Apple Pay，集成简单。
  • 适用场景：网页端或插件支付功能。

2.4、建站与设计

  1. WordPress
  • 特点：开源 CMS，插件生态丰富，适合快速搭建内容型网站。
  • 适用场景：博客、电商、企业官网。
  1. Figma https://www.figma.com/
  • 特点：在线协作设计工具，支持原型制作。
  • 适用场景：UI/UX 设计，团队协作。

2.5 图片服务器

  • ImgBB ：https://imgbb.com/ 是最好的免费图片托管服务之一，无需注册就能使用。支持 JPEG、PNG、BMP、TIFF、GIF、WEBP、HEIC、PDF 等多种格式，上传文件的最大限制为 32MB，不限制上传图片数量和网上停留时间。上传后可快速获取全尺寸图像直接链接或缩略 HTML 链接，方便在博客中使用。
  • Postimage ：https://postimg.cc/files 无需注册，成立于 2004 年，运行稳定。支持 JPEG 和 PNG 等文件格式，可上传最大 8MB 大小的图像。用户能自定义分享图片大小，设置永久链接或自动失效链接，有效期最长 31 天，适合在网站、博客、留言板和论坛中使用。（优点：会生成各个渠道的超链）

3.运维

3.部署与运维

  1. Vercel
  • 特点：静态网站托管，支持 Serverless 函数，一键部署。
  • 适用场景：前端项目快速上线，适合 React/Vue 等框架。
  1. Netlify
  • 特点：免费 CDN 加速，自动化构建与部署，适合静态站点。
  • 适用场景：博客、营销页面等轻量级网站。
  1. Docker
  • 特点：容器化部署，简化环境配置。
  • 适用场景：跨平台应用打包与发布。

3.2 数据分析与监控

  1. Google Analytics (GA)
  • 特点：免费且功能强大，追踪用户行为。
  • 适用场景：网站流量分析。
  1. Umami
  • 特点：开源、隐私友好的分析工具，支持实时数据。
  • 适用场景：注重用户隐私的项目。
  1. Prometheus + Grafana
  • 特点：开源监控方案，实时跟踪应用性能。
  • 适用场景：服务器与微服务监控。

工具

实用工具

  • AWS SES/Resend：邮件服务，支持交易与营销邮件。

工作

1. 核心成果回顾

  • 调整：元旦过后第1天，团队再次迎来调整； 年中时，团队再次调整；1年2次调整，换了2个老板；
  • 关键项目完成情况：通过nova选品、活动补贴、M2A等项目，在完成技术架构升级的同时，很好的做到业务赋能，提升团队效率。
  • 绩效：高绩效，但无晋升机会(业务下滑+老板调整)；

2. 存在问题反思

  • 工作中的不足：更多的是应对变化和日常项目落地，缺少积累；
  • 流程/方法问题：团队调整意味着需要和新的老板重新对焦，建立信任；疲于应对混乱的日常事务，缺少积累。

2023年总结

Michael Song 发布于 2024-01-01 收录于 总结

核心主旨：复盘成长轨迹，沉淀经验方法，明确未来方向

重点突出:具体成果、可落地改进措施，避免空泛表述

工作

1. 核心成果回顾

  • 调整：年初部门调整，从AE调整到Lazada，做的业务还是面向欧洲用户的电商，工作沟通依旧需要倒时差，日常沟通及大促值班时很酸爽；
  • 关键项目完成情况：快速熟悉基于Lazada的Miravia活动招商系统，同时参考AE国际化技术链路进行简要优化重构；
  • 能力提升突破：从0-1，快速熟悉了解业务和系统，成为领域PIC，形成个人方法论，优化跨团队合作流程。
  • 创办公益幸福团，组织活动，传递价值。

2. 存在问题反思

  • 工作中的不足：英语沟通薄弱，影响效率；
  • 流程/方法问题：创新团队跨团队沟通成本高，多任务并行时优先级混乱；疲于应对混乱的日常事务，影响个人精进。
  • 外部因素影响：客观团队环境变化频繁，年度更换3个老板，总体应对不足，被动接受组织调整。

3. 下一年度

  • 核心目标：取得高绩效，晋升。
  • 具体措施：制定学习计划、优化沟通协作机制。

学习

1. 学习成果

  • English：从内网中找到1家online课程，老师是印尼人，虽然口音有点差，但性价比高，通过每日25分钟对话，在完成2个月英语在线课程后，敢于开口讲述。
  • book：读了6本书，阅读的终局是只读经典，理解文字及文字背后的底层逻辑、社会运转的规律，遵循规律。

生活

1. 年度生活亮点

  • 重要事件：结婚；
  • 年底在老家办的婚礼，辛苦爹妈操劳；现在操办婚礼比小时候方便很多，会有专门的婚庆公司可以帮你搞定一些你不懂的流程、事项、装备等，酒席也是在专门的酒店安排，不用在家办，可以省很多时间和麻烦事。

2. 生活中的困扰

  • 平衡问题：工作与生活失衡，陪伴家人时间几乎没有，因为是新的团队+创业业务，周末也会频繁加班；更多的时间被工作所占用，导致生活质量下降。

3. 下一年度

  • 多陪陪家人

健康

1. 年度健康状况

  • 身体状态：轻度脂肪肝倾向，需要减重和运动。
  • 总运动10次，累计146km；其中爬山：7次，累计63km；骑行3次，累计83km；

2. 下一年度

  • 运动目标：每月1次爬山，年度80km以上累计跑步里程

旅行

整体复盘与展望

  • 利用一切时间和机会，持续精进自己，让自己有较高的抗风险能力。

  • 用心感悟人生、生活。

Michael Song 发布于 2022-12-10 收录于 读书笔记

关于书

  • 《断舍离.心灵篇》
  • 山下英子; 王珊译
  • 广西科学技术出版社

三种扔不掉东西的人

逃避现实型

  • 真麻烦；
  • 这种类型的人太忙碌，几乎没什么时间待在家里，所以也没办法收拾屋子。多数情況下，这种人都是对家庭有所不满， 不愿意待在家里，所以就找各种各样的事，让自己忙碌起来。加上家里乱七八糟的，所以就更不想待在家里了。慢慢地，在这样的恶性循环里越陷越深。

执着过去型

  • 真可惜；
  • 这种类型的人，即便是现在己经不会再用的过去的东西，也非得收着不可。相册、奖杯等等，统统当作命根子似的保管起来。 他们多半隐含着对过去幸福时光的留恋。从不想面对现实这一层意义上来看，与逃避现实型也有相通之处。

担忧未来型

  • 也许有天会用到；
  • 这种类型的人致力于投资不知何时会发生的未来的不安要素。这类人的特点是过分地囤积纸巾等日用品，要是没了这些就会觉得困扰、焦虑不安。 在三种类型当中，这一类的人最多。

效率

  • 效能，单位时间内产生的生产价值；提高效能意味着 使用相对少的时间，产生相对大的生产价值。

  • 效率，在产生效能过程中的使用的方式方法等。

一、数据结构的存储方式

数据结构的存储方式只有两种：数组（顺序存储）和链表（链式存储）

这句话怎么理解，不是还有散列表、栈、队列、堆、树、图等等各种数据结构吗？

我们分析问题，一定要有递归的思想，自顶向下，从抽象到具体。你上来就列出这么多，那些都属于「上层建筑」，而数组和链表才是「结构基础」。因为那些多样化的数据结构，究其源头，都是在链表或者数组上的特殊操作，API 不同而已。
